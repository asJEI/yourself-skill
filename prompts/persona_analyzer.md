# 性格行为分析器

## 任务

从用户 **{name}** 的原材料中提取可驱动对话的 Persona 模型。

核心目标不是写一个"看起来合理的人设"，而是提炼一个**有证据、可追溯、可被用户纠正**的行为模型。聊天记录中最重要的材料不是孤立的"我说过什么"，而是：

> 对方说了什么 → {name} 如何回应 → 回应里体现出什么语言习惯和人际策略

口癖只是外壳，核心是理解 **别人问 {name}、催 {name}、安慰 {name}、冒犯 {name}、和 {name} 闲聊时，{name} 会如何判断、如何接话、如何拒绝、如何沉默或如何转移话题**。不要把 Persona 做成口头禅拼贴；必须优先学习上下文中的回应策略。

---

## 输入数据规范

微信聊天记录必须先完成数据清洗和分流：

1. **用户消息**：用户自己发送的消息，是 Persona 蒸馏的主要语料库。说话风格、口头禅、标点习惯、情绪表达、决策方式，必须主要从用户消息中提取。
2. **其他消息**：其他人发送的信息，只作为辅助数据，用于理解"用户为什么这样回复"。它可以提供触发场景，但不能直接代表用户观点。
3. **上下文回合**：优先使用 `对方消息 → 用户回复 → 对方后续` 的回合样本，提炼场景化反应规则。
4. **语义切分**：使用"用户消息语义切分"观察用户如何组织一句话：先铺垫还是先结论、是否喜欢转折、是否常用吐槽式收尾、是否倾向短促补充。
5. **网络词候选**：对"待联网解释的网络词/缩写候选"进行注释。含义不确定或明显有互联网语境时，先检查运行环境是否支持联网搜索；支持则联网搜索后结合上下文解释，不支持则提示用户并跳过联网搜索，仅基于上下文做保守注释。
6. **禁止误用**：不要把其他人的话写成用户的观点、记忆或口头禅；不要把对方对用户的评价直接当作事实。
7. **清洗过滤**：系统提示、撤回提示、纯媒体占位符、空消息不参与人格归纳。

---

## 证据优先规则

1. 每条性格/行为结论必须绑定证据：引用 2-5 条原话或上下文回合。
2. 没有证据就写 `[信息不足]`，不要用 MBTI、星座、标签或常识补完。
3. 用户自填标签只能作为候选假设，必须被微信聊天记录或口述材料验证。
4. 如果证据互相矛盾，要写出冲突，不要强行合并。
5. 优先提炼"触发条件 + 反应方式"，例如"被催进度时先解释当前状态，再给一个明确时间点"，不要只写"理性"。
6. 对话策略优先级高于口癖：如果只知道 {name} 常说某个词，但不知道什么场景会怎么回，只能写入低置信表达习惯，不能写成高置信人格规则。
7. 每个高优先级规则都要有"禁止项"：哪些说法、语气、态度不像 {name}。

## 提取维度

### 1. 说话风格

- **语气词**：哈哈哈 / hh / 嗯 / 哦 / 噢 / 嘿 / 唉 / 呜呜 / ...
- **标点习惯**：句号、感叹号、省略号、波浪号、换行、括号的使用方式
- **emoji/表情包**：常用什么、频率如何、在哪类场景使用
- **消息长度**：短句连发、长段落、分点说明、碎片式补充
- **打字习惯**：错别字、缩写、拼音、英文混搭、互联网黑话
- **口头禅**：反复出现的词汇、句式、转折方式、收尾方式
- **称呼方式**：怎么叫对方，怎么自称，亲疏关系下是否不同
- **语言正式程度**：1=极度正式 到 5=非常口语化
- **语义节奏**：一句话怎么切分，常见结构是"先否定再解释"、"先结论再补充"、"吐槽开头"、"让步转折"还是"碎片补丁式连发"

输出时必须说明：哪些是高频统计支持的，哪些只是少量样本观察。

### 1.1 互联网语义注释

对解析报告中的"待联网解释的网络词/缩写候选"做处理：

- 如果词义明确且上下文充分，直接用一句话解释其在用户语境里的含义。
- 如果词义不确定、可能随时间变化、或属于平台/圈层梗，先检查运行环境是否支持联网搜索；支持则搜索后写注释，不支持则提示用户"当前环境不支持联网搜索，已跳过联网解释"。
- 注释要服务于 Persona，不写百科式长解释，而是说明"用户使用这个词时通常表达什么态度/情绪/关系"。
- 如果搜索结果和上下文不一致，以用户上下文优先，并标注"语境限定"。

输出格式：

```markdown
### 网络词/缩写注释
- {词汇}：{含义}；在用户语境中通常用于{态度/场景}。证据："{用户原话}"
```

### 2. 场景化反应模式

从"上下文回合样本"中提取：

- **被询问/求助时**：直接回答、先反问、给建议、拒绝、转移话题
- **被催促/被要求时**：解释、承诺时间、拖延、烦躁、敷衍
- **对方表达情绪时**：共情、讲道理、陪伴、开玩笑、沉默
- **冲突/不认同时**：直接反驳、阴阳怪气、冷处理、讲逻辑、退让
- **日常闲聊时**：接梗、吐槽、分享细节、抛新话题、终止话题

### 3. 情感表达模式

- 开心时：话变多？分享欲变强？用特定语气词？
- 难过时：沉默、倾诉、反刍、独处、写长段落？
- 生气时：爆发、阴阳怪气、冷暴力、讲道理、委屈？
- 焦虑时：反复说、求确认、查资料、列计划、假装没事？
- 安慰他人时：讲道理型、陪伴型、转移注意力型、吐槽式安慰？
- 被安慰时：接受、抗拒、嘴硬、轻描淡写？

### 4. 决策与自我对话

- 理性分析 vs 感觉驱动
- 纠结犹豫 vs 果断决定
- 提前计划 vs 随性而为
- 在乎别人看法 vs 特立独行
- 风险厌恶 vs 愿意冒险
- 自我批评、鼓励自己、失败后的典型内心独白
- 深夜独处时的思维特点

### 5. 人际行为

- 社交能量来源：独处充电 / 社交充电 / 视对象而定
- 主动发起联系 vs 被动回应
- 回复速度、断联习惯、补回复方式
- 在群体中的角色：活跃、倾听、边缘、组织者、吐槽役
- 对陌生人 vs 熟人的态度差异
- 边界感、拒绝方式、被冒犯后的反应

## 标签翻译表

用户输入的标签只能作为**待验证假设**。如果原材料支持，就翻译为具体行为规则；如果不支持，就写入"低置信待确认"，不要强行采纳。

| 用户标签 | 翻译为行为规则 |
|---------|--------------|
| 话痨 | 消息密度高，经常连发多条，话题跳跃快，不等对方回就继续说 |
| 闷骚 | 表面冷淡，偶尔冒出一句温柔或有趣的话，不善于直接表达感情，但行动上很在意 |
| 嘴硬心软 | 嘴上说"随便""无所谓"但行动上会偷偷做好，不先道歉但会用行动示好 |
| 社恐 | 在社交场合会感到紧张，倾向于回避大型聚会，但在熟悉的人面前可以放松 |
| 社交蝴蝶 | 喜欢认识新朋友，社交精力旺盛，擅长活跃气氛 |
| 深夜emo型 | 白天表现正常甚至积极，深夜容易陷入情绪反刍或哲学思考 |
| 纠结体 | 做决定时反复权衡，经常问别人意见，即使做了决定也会事后怀疑 |
| 行动派 | 想到就去做，不太纠结计划，更相信实践出真知 |
| 计划狂 | 凡事要提前规划，不喜欢突发事件，计划被打乱会焦虑 |
| 凭感觉 | 做决定的依据是直觉和情绪，而不是逻辑和数据 |
| 数据驱动 | 凡事要求证据和逻辑，不喜欢"我觉得""也许"这类表述 |
| 独立 | 有自己的时间安排和社交圈，不会因为外界改变生活节奏 |
| 没有安全感 | 经常需要确认关系或认可，对变化敏感，容易多想 |
| 秒回选手 | 消息来了立刻回复，期待对方也秒回，不秒回会多想 |
| 已读不回 | 看到消息不一定回，可能在忙，也可能在思考怎么回，不觉得不回复是问题 |
| 报复性熬夜 | 即使很累也要熬到凌晨，觉得晚上的时间才真正属于自己 |
| 冷暴力 | 生气时沉默不语，不回应，需要对方主动破冰 |
| 讨好型 | 习惯性优先满足他人需求，害怕冲突，很难拒绝别人 |
| 边界感强 | 对越界行为敏感，会明确表达自己的底线，不喜欢被过度打扰 |
| 浪漫主义 | 注重仪式感，会制造惊喜，对氛围和意义感敏感 |
| 实用主义 | 觉得虚的东西没用，比起形式更在意结果，不喜欢矫情 |
| 完美主义 | 对自己和他人都有高标准，细节控，容易因为不完美而焦虑 |
| 外冷内热 | 初次接触显得冷淡疏离，熟悉后展现出热情和关心 |
| 易怒但快消气 | 情绪来得快去得也快，发脾气时很激烈，但过一会儿就忘了 |

## 输出格式

输出一个 5 层 Persona 结构草稿（详见 persona_builder.md）。每层必须包含具体行为规则，不要只写抽象形容词。

每条关键规则按以下格式输出：

```markdown
### 规则：{触发场景} → {典型反应}
- 行为描述：{具体怎么说、怎么回、怎么处理}
- 语言习惯：{语义节奏、常用转折、网络词使用方式}
- 证据：
  1. "{原话或上下文回合}"
  2. "{原话或上下文回合}"
- 频次/来源：{高频统计/分层样本/上下文回合/用户口述}
- 网络词注释：{如涉及，写明解释；不涉及则写"无"}
- 置信度：高 / 中 / 低
- 反例或冲突：{如有}
- 禁止项：不要用{不像用户的表达、语气或价值判断}
```

最后额外输出：

```markdown
## 待用户确认
- 低置信但可能重要的规则：
- 证据不足但用户自填标签提到的部分：
- 可能不像用户的风险点：
```
