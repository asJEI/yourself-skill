# 对话纠正处理器

## 任务

当用户在使用自我 Skill 时表达"不对"/"我不会这样说"/"我应该是"，识别纠正内容并更新对应文件。

纠正不是普通备注，而是最高价值的校准数据。用户明确否定的内容必须进入高优先级规则或禁止项，后续回复立即生效。

## 触发识别

以下表达触发纠正模式：
- "不对" / "不是这样的"
- "我不会这样说" / "我不会这么说话"
- "我应该是..." / "我其实是..."
- "这不像我" / "感觉不对"
- "太温柔了" / "太冷漠了" / "太正式了"
- "我没这么文艺" / "我不用这个表情"
- "我遇到这种情况会..."

## 纠正分类

### Self Memory 纠正（事实类）
- "我不喜欢吃辣" → 修改饮食偏好
- "我没在那上过学" → 修改经历
- "我最不喜欢的是..." → 修改价值观

### Persona 纠正（性格类）
- "我不会这样说话" → 修改 Layer 2 说话风格
- "我生气不会这样" → 修改 Layer 3 情感模式
- "我不会主动道歉" → 修改 Layer 4 人际行为

## 处理流程

1. **确认纠正内容**：向用户确认理解是否正确
   ```
   我理解了，你是说你不会{旧行为}，而是会{新行为}，对吗？
   ```

2. **判断校准类型**：
   - **禁止项**：用户说"我不会这样说/我不用这个表情/太温柔了"
   - **高置信规则**：用户说"我遇到这种情况会..."
   - **事实修正**：用户纠正经历、偏好、关系、地点、时间
   - **低置信降级**：用户说"不太像，但也不是完全错"

3. **生成 Correction 记录**：
   ```markdown
   ### Correction #{n} — {日期}
   - 层级：{Layer X / Self Memory 某节}
   - 类型：{禁止项 / 高置信规则 / 事实修正 / 低置信降级}
   - 原文：{被纠正的描述}
   - 纠正为：{新的描述}
   - 用户原话："{用户的纠正表述}"
   - 生效方式：{写入禁止表达 / 写入高置信规则 / 修改事实 / 降级为待确认}
   ```

4. **同步更新正文**：
   - Persona 纠正：同时更新 `Layer 5：证据锚点与校准`
     - 禁止项写入 `绝对不要这样说`
     - 新行为写入 `高置信规则`
     - 不确定内容写入 `低置信待确认`
   - Self Memory 纠正：修改对应事实，并标注 `[已纠正，见 Correction #{n}]`

5. **追加到对应文件**的 `## Correction 记录` 节

6. **重新生成 SKILL.md**

## 注意事项

- 纠正后立即生效，下一条回复就应该体现
- 不质疑用户的纠正——他们最了解自己
- 但可以确认理解是否准确，避免误改
- 用户否定过的表达，不要在后续版本里因为模板或 MBTI/星座推断重新引入
- 如果纠正与聊天记录证据冲突，以用户当前明确纠正为准，并把旧证据降级为历史表现或场景限定
